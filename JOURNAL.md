| 4 | 2026-03-28 | Rate Limiter | O(1) to O(log n) | Strategy, Factory, Builder, Registry | Implemented 5 rate limiting algorithms (Token Bucket, Leaky Bucket, Fixed Window, Sliding Window Log, Sliding Window Counter) as interchangeable strategies. Factory Pattern centralizes creation logic. Thread-safe implementations using ReentrantLock and AtomicLong. Registry Pattern enables per-user rate limiting with lazy initialization. Key insight: Sliding Window Counter provides best balance (O(1) time/space, high accuracy, no boundary issues) for production use. Token Bucket allows bursts, Leaky Bucket smooths traffic, Fixed Window has 2x boundary problem, Sliding Window Log offers perfect accuracy at O(n) space cost. |
| 5 | 2026-04-14 | Task Scheduler | O(log n) | Builder, Adapter, Facade, Producer-Consumer | Production-grade scheduler using DelayQueue for priority task ordering. Supports one-time (delayed/absolute), recurring (fixed rate/delay), and wall-clock time scheduling. Dispatcher thread pattern with worker pool for execution. Builder Pattern enables fluent task configuration. Adapter Pattern (DelayedTask) integrates with DelayQueue. Key insight: Single dispatcher thread + worker pool separates scheduling from execution, enabling precise timing with concurrent execution. Task lifecycle management with atomic state transitions (SCHEDULED→RUNNING→COMPLETED/CANCELLED). |
| 6 | 2026-04-17 | Order Book Engine | O(log n) | Thread Confinement, Strategy, Producer-Consumer, Facade | Per-symbol SingleThreadExecutor eliminates all locking inside the matching engine — thread confinement provides serial safety without synchronized/ReentrantLock. Each symbol (AAPL, AMZN) owns an isolated OrderBook with plain (non-concurrent) TreeMap<price, Deque<Order>> for price-time priority: O(log n) price access + FIFO within a price level. A flat PriorityQueue cannot preserve time priority within the same price. ConcurrentHashMap is the only shared structure, used solely for computeIfAbsent symbol registration. TradeListener (Strategy) fully decouples event handling from matching logic. |
